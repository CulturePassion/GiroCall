07 — Spacing Fixes

Before / After

Polish gaps identified in v1 implementation. These are the non-negotiable spacing corrections.

Before — Warning banner placement

Config warning appeared in different positions at 900px breakpoint — left column on desktop, top on mobile.

Inconsistent · confusing on resize

After — Unified top banner

Warning always at page top, full-width, above all layout splits. 16px gap to content.

✓ Fixed in login_screen.dart

Before — Touch targets 44dp

AppTokens.minTouchTarget = 44 — below Material 3 / HIG 48dp minimum.

After — Touch targets 48dp

AppTokens.minTouchTarget = 48 — all CTAs, nav items, icon buttons.

Before — Wheel cramped on narrow

Wheel margin < 16px on small screens. CTA buttons full-bleed without max-width.

After — Wheel breathing room

Min 40px margin around wheel. CTAs max 280px centered. Result card max 340px.

09 — Accessibility

Inclusive Design

High contrast, readable text, proper focus states. Never sacrifice clarity for aesthetics.

Contrast Ratios (WCAG AA)

  • Text on light bg: 12.6:1 (#1E293B on #F8FAFC)
  • Text on dark bg: 15.2:1 (#F1F5F9 on #0F172A)
  • Teal on white: 4.6:1 (large text OK)
  • Coral button text: 4.5:1 (white on #F97316)

Focus & Semantics

  • 2.5px teal focus ring on all interactive elements
  • Semantic labels on wheel ("Spin to select a contact")
  • Live regions for spin result + rating feedback
  • 48dp minimum touch targets
  • Respects prefers-reduced-motion

Dark Mode First

System default with local persistence. Hero screens (Giro, Auth) use gradient backgrounds. Content screens use calm dark surfaces (#0F172A / #1E293B). The wheel's vibrant slices provide natural color pop without overwhelming.
